Hexo同时部署到GitHub和VPS

1. hexo部署到VPS

安装Git

apt-get install git

配置 SSH 免密访问

  • 本地安装 Git

  • 配置本地免密ssh登陆到远程VPS

    先检查是否有rea文件

    • 进入 ~/.ssh/ 目录

    • 创建rsa 

    • ssh-keygen -t rsa

    • 一路回车，会在 ~/.ssh/ 目录下生成 id_rsa 和 id_rsa.pub 两个文件

    • 把生成的 id_rsa.pub 文件内容 copy到 VPS 的 root用户的 ~/.ssh/authorized_keys文件中

  • 客户端免密登录到 VPS 

    ssh root@ip

    • 如果出现不能登录的情况，可能是文件权限有问题，VPS 上做如下修改：

    • chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

在 VPS 上创建仓库

cd ~
mkdir hexo.git
cd hexo.git
git init --bare

Git Hook 自动 Clone 收到提交的仓库到网站根目录

VPS的 hexo网站目录为 /root/wwwroot/hexo/

建立一个仓库内容转移的缓存目录：

mkdir -p /root/tmp/hexo_tmp

然后处理 hexo.git 提交的事件，自动更新内容到 hexo网站目录。

vi /root/hexo.git/hooks/post-receive

编辑 post-receive 文件：

#!/bin/bash -l
GIT_REPO=/root/hexo.git
TMP_GIT_CLONE=/root/tmp/hexo_tmp
PUBLIC_HEXO=/root/wwwroot/hexo
rm -rf ${TMP_GIT_CLONE}
git clone $GIT_REPO $TMP_GIT_CLONE
rm -rf ${PUBLIC_HEXO}/*
cp -rf ${TMP_GIT_CLONE}/* ${PUBLIC_HEXO}

脚本可以实现完全更新，即先删除原网站内所有内容，再拷贝新仓库内容到网站目录

给脚本加上执行权限：

chmod +x /root/hexo.git/hooks/post-receive

多部署(vps+github)

配置 _config.xml

deploy:
- type: git
  repository:  git@github.com:Evan-ZJ/Evan-ZJ.github.io.git
  branch: master
- type: git
  repository:  root@39.106.97.116:hexo.git
  branch: master

2. nginx安装与设置

  • 安装nginx
sudo apt install nginx
  • 向 nginx 写入配置，将/root/wwwroot/hexo作为主目录
cd /etc/nginx/sites-available 
vi default
  • 内容删除，写上：
server {
        listen 80;
        listen [::]:80;

        root /root/wwwroot/hexo;
        index index.html index.htm index.nginx-debian.html;

        server_name 域名;#如果没有的话就别填

        location ~* ^.+\.(ico|gif|jpg|jpeg|png)$ {
                root /var/www/hexo;
                access_log   off;
                expires      1d;
        }
        location ~* ^.+\.(css|js|txt|xml|swf|wav)$ {
                root /var/www/hexo;
                access_log   off;
                expires      10m;
        }
        location / {
                root /var/www/hexo;
                if (-f $request_filename) {
                        rewrite ^/(.*)$  /$1 break;
                }
        }
}

保存退出，然后使用

service nginx restart

3. 错误解决

多rsa问题

  • 有时候本地的 ~/.ssh 目录中有多个自己生成的 id_rsa.pub文件;

  • 这些 id_rsa.pub 文件是基于相同的用户名和邮箱生成的

    ssh 可以登陆，但在 hexo 部署时报错，错误是远程权限拒绝;

执行 hexo d命令时，默认使用的不是添加到 VPS中的~/.ssh/authorized_keys的Key的对应的 id_rsa.pub

在本地~/.ssh目录中，（添加一个名为config的文件，这样写：

Host 106.186.33.133
    HostName        ip
    User            用户名
    IdentityFile    .ssh/id_rsa

清空known_hosts，然后重新连接部署

Nginx 出现 403 Forbidden

目录权限
chmod -R 755 /root/wwwroot/hexo
打开nginx.conf
vi /etc/nginx/nginx.conf

把user 用户名改为 user root
